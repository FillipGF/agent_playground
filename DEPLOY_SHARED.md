# Инструкция по деплою совместного отчета на хостинг Timeweb

Мы подготовили бэкенд на Python (Flask) с базой данных MySQL, чтобы задачи, канбан-доски и решения по станциям синхронизировались в реальном времени между всеми пользователями.

## 1. Загрузка файлов на сервер
Подключитесь к вашему серверу через SFTP/FTP клиент (например, FileZilla) или файловый менеджер Timeweb и скопируйте файлы вашего проекта в рабочую директорию на сервере:
- Файл `План развития городов_Июнь.html` (в корень приложения)
- Файл `config.json` (в корень приложения)
- Папку `fleet/` (включая файлы `fleet/server.py` и `fleet/generate_city_dev_plan.py`)

## 2. Установка зависимостей на сервере
Для работы бэкенда необходимы следующие библиотеки Python:
- `Flask` (веб-сервер)
- `Flask-SQLAlchemy` (ORM для работы с БД)
- `PyMySQL` (драйвер подключения к MySQL)
- `cryptography` (для безопасной аутентификации в MySQL)

Вы можете установить их на сервере командой:
```bash
pip install flask flask-sqlalchemy pymysql cryptography
```
*(Или через панель управления Timeweb в разделе настройки Python-окружения вашего приложения).*

## 3. Запуск сервера на Timeweb
В панели управления Timeweb вы можете настроить запуск Python-приложения:
1. Создайте **Python-приложение** в панели (или настройте службу / systemd-сервис, если это VPS).
2. Укажите путь к файлу запуска: `fleet/server.py`.
3. Укажите порт (по умолчанию `5000` или настройте проксирование через Nginx).

### Пример настройки автозапуска через systemd (для VPS):
Создайте файл сервиса `/etc/systemd/system/city-plan.service`:
```ini
[Unit]
Description=City Dev Plan Shared Dashboard
After=network.target

[Service]
User=root
WorkingDirectory=/root/Agent_Playground
ExecStart=/usr/bin/python3 fleet/server.py
Restart=always

[Install]
WantedBy=multi-user.target
```
Запустите службу командами:
```bash
systemctl daemon-reload
systemctl start city-plan
systemctl enable city-plan
```

---

## 4. Проверка
После запуска перейдите в браузере по адресу вашего сервера (например, `http://ваш-домен.ru:5000` или настроенному домену).
- Все выбранные решения («Демонтировать», «Проаккаунтить», «Ожидание B2B») теперь сохраняются в базу данных MySQL.
- Задачи и канбан-доски будут синхронно отображаться у всех пользователей, открывших этот сайт.
