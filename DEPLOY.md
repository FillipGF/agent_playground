# Инструкция по переносу и запуску скриптов на сервере (Ubuntu / Linux)

Для того чтобы скрипты автоматизации (`automate_tasks.py` и `sync_operations.py`) запускались автоматически 24/7, их нужно перенести на ваш локальный сервер.

---

## Шаг 1. Подготовка файлов и перенос профиля браузера

Так как на сервере нет графического интерфейса (GUI) для ввода логина и пароля Яндекса, мы перенесем авторизованную сессию с вашего компьютера в виде сжатого архива.

1. **На вашем рабочем компьютере** запустите скрипт для упаковки профиля:
   ```bash
   uv run python fleet/pack_profile.py pack
   ```
   В результате в папке `fleet/` создастся компактный файл `chrome_profile.zip` (кеш, временные файлы и логи будут автоматически исключены для экономии веса).

2. **Перенесите файлы проекта на сервер.** Это делается через SCP (с использованием вашего SSH-ключа):
   ```bash
   scp -r fleet telegram_bot config.json service_account.json switch@192.168.0.178:/home/switch/rabota/
   ```

3. **На сервере** распакуйте профиль:
   ```bash
   cd /home/switch/rabota
   python3 fleet/pack_profile.py unpack
   ```
   Скрипт воссоздаст папку `fleet/.chrome_profile` с уже готовой авторизацией.

---

## Шаг 2. Установка окружения на сервере

На сервере используется системный Python 3. Все библиотеки устанавливаются в локальный профиль пользователя `switch` через `pip` (без использования виртуальных окружений и `sudo`):

1. **Установите библиотеки (если появятся новые зависимости):**
   ```bash
   python3 -m pip install --user pandas numpy gspread google-auth requests openpyxl playwright pymysql
   ```

2. **Установите Chromium для Playwright:**
   Если браузер Playwright еще не скачан в профиль, выполните:
   ```bash
   python3 -m playwright install chromium
   ```

---

## Шаг 3. Проверка запуска вручную

Проверьте, что синхронизация работает в фоновом (headless) режиме с использованием перенесенного профиля:

1. **Запуск сбора и обработки текущих задач (Google Sheets):**
   ```bash
   python3 fleet/automate_tasks.py --download
   ```

2. **Запуск синхронизации выполненных задач в базу MySQL:**
   ```bash
   python3 fleet/sync_operations.py
   ```

*Благодаря оптимизации проверки дубликатов в памяти (без Full Table Scan), импорт происходит практически мгновенно.*

---

## Шаг 4. Настройка автоматического запуска по расписанию (cron)

Чтобы скрипты выполнялись самостоятельно в нужное время, настройте планировщик задач `cron`.

1. Откройте редактор cron на сервере:
   ```bash
   crontab -e
   ```

2. Добавьте задачи в конец файла:

   ```cron
    # 1. Автоматическая выгрузка и обновление кэша норм заполняемости (раз в день в 03:00)
    0 3 * * * cd /home/switch/rabota && /usr/bin/python3 fleet/sync_fullness_norms.py >> /home/switch/rabota/cron_norms.log 2>&1

    # 2. Автоматическое скачивание и обновление задач в Google Sheets (в 08:30, 14:30 и 20:30)
    30 8,14,20 * * * cd /home/switch/rabota && /usr/bin/python3 fleet/automate_tasks.py --download >> /home/switch/rabota/cron_tasks.log 2>&1

    # 3. Синхронизация выполненных задач в MySQL (в 08:45, 14:45 и 20:45)
    45 8,14,20 * * * cd /home/switch/rabota && /usr/bin/python3 fleet/sync_operations.py >> /home/switch/rabota/cron_sync.log 2>&1
    ```

3. Сохраните и закройте файл. Теперь задачи будут автоматически запускаться в фоновом режиме по расписанию. Вы сможете проверять статус выполнения в файлах логов `cron_norms.log`, `cron_tasks.log` и `cron_sync.log`.

## Обновление авторизации (если сессия истечет)

Яндекс периодически сбрасывает сессии авторизации (обычно раз в несколько месяцев). Если в логах появится ошибка `Необходима авторизация`:
1. На вашем компьютере запустите:
   ```bash
   uv run python fleet/login_and_inspect.py
   ```
2. Войдите в аккаунт Яндекса в открывшемся окне браузера и нажмите ENTER.
3. Снова упакуйте профиль: `uv run python fleet/pack_profile.py pack`.
4. Перенесите `chrome_profile.zip` (созданный в папке `fleet/`) на сервер и распакуйте: `python3 fleet/pack_profile.py unpack`.
